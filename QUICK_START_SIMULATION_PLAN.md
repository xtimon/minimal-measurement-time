# Быстрый старт: План симуляций

## Что было создано

1. **`simulation_plan.py`** - Скрипт для генерации и выполнения плана симуляций
2. **`analyze_simulation_results.py`** - Скрипт для анализа результатов
3. **`SIMULATION_PLAN_README.md`** - Подробная документация

## Быстрый запуск

### 1. Создание и выполнение плана симуляций

```bash
python simulation_plan.py
```

Скрипт:
- Создаст план симуляций для всех сред (фермионы, бозоны, классические)
- Сохранит план в JSON файл
- Спросит, сколько симуляций выполнить (для теста введите 50-100)
- Выполнит симуляции и сохранит результаты

### 2. Анализ результатов

```bash
# Сначала установите matplotlib (опционально, для графиков)
pip install matplotlib

# Затем запустите анализ
python analyze_simulation_results.py results/simulation_plan/final_results_YYYYMMDD_HHMMSS.json
```

## Что включено в план

### Параметры для всех сред:
- **Количество бит (ΔI)**: 1, 2, 4, 8, 16, 32, 64, 128, 256, 512, 1024
- **Количество частиц (N)**: 10², 10³, 10⁴, 10⁵, 10⁶, 10⁷
- **Температура системы (T)**: 0.01, 0.1, 1.0, 10.0, 100.0, 300.0, 1000.0 K

### Специфичные параметры:

**Фермионы:**
- Температура Ферми (T_F): 100, 500, 1000, 5000, 10000, 50000 K
- Критическая температура (T_c): 10, 50, 100, 500, 1000, 5000, 10000 K

**Бозоны:**
- Критическая температура (T_c): 0.001, 0.01, 0.1, 1.0, 10.0, 100.0 K
- Температура конденсации (T_c_bose): обычно равна T_c

**Классические:**
- Критическая температура (T_c): 10, 50, 100, 300, 500, 1000 K

### Дополнительные параметры:
- Энергия взаимодействия (U): 0.0, 0.1, 0.3, 0.5, 0.7, 0.9
- Ширина зоны (W): 0.5, 1.0, 2.0, 5.0
- Времена декогеренции (T1, T2): различные значения

## Результаты

После выполнения вы получите:

1. **JSON файл с планом** - `results/simulation_plan/simulation_plan_*.json`
2. **JSON файл с результатами** - `results/simulation_plan/final_results_*.json`
3. **Текстовый отчет** - `results/simulation_plan/final_results_*_summary.txt`
4. **Графики** (если установлен matplotlib):
   - Распределение по типам систем
   - Зависимость от количества бит
   - Зависимость от количества частиц
   - Зависимость от температуры
   - Корреляции между параметрами
5. **Детальный отчет** - `results/analysis/analysis_report_*.md`

## Советы

1. **Для тестирования**: Запустите с ограничением 50-100 симуляций
2. **Для полного анализа**: Выполните все симуляции (может занять несколько часов)
3. **Используйте параллельную обработку**: По умолчанию используются все ядра CPU для ускорения
4. **GPU vs CPU**: 
   - CPU с параллельной обработкой обычно быстрее для больших планов
   - GPU работает последовательно, но может быть быстрее для отдельных симуляций
5. **Промежуточное сохранение**: Результаты сохраняются каждые 100 симуляций автоматически

## Пример использования

```python
from simulation_plan import SimulationPlan

# Создание плана
plan = SimulationPlan()
plan.generate_plan()
plan.save_plan()

# Выполнение с параллельной обработкой (использует все ядра CPU)
plan.execute_plan(max_simulations=100, use_gpu=False, n_jobs=None)

# Или с указанием количества процессов
plan.execute_plan(max_simulations=100, use_gpu=False, n_jobs=8)

# Сохранение результатов
plan.save_final_results()
```
